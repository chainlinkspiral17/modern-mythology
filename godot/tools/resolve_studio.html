How to run the actions

1. Start Resolve. Enable Preferences → System → General → External scripting using: Local.

2. Build actions here. Export actions.json.

3. Run:

python3 godot/tools/resolve_bridge.py actions.json --dry-run
python3 godot/tools/resolve_bridge.py actions.json

Or, skip the JSON entirely and import a Runway manifest directly:

python3 godot/tools/resolve_bridge.py \
  --from-runway-manifest godot/assets/video/generated/manifest.json \
  --volume vol5 \
  --timeline

If the env vars aren't set, the bridge auto-bootstraps from /opt/resolve.

The action list runs in order against a running Resolve. Default flow: ensure project → import Runway manifest → build a timeline → sync metadata back. The sync step is the loop-closer — it pulls markers, comments, and flag colors back into the manifest under a resolve.* subkey so the rest of the deck's tooling knows which Runway clips have been graded or flagged.

↩ Sync metadata back into the manifest

For each clip in the project that matches a manifest entry, pull markers, Comments, and flag color back into the manifest under resolve.*. This is what closes the loop — once you've graded/marked clips in Resolve, run this to let the rest of the deck's tools see your decisions.
