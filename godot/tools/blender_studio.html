THE PIPELINE

  ┌────────────────────────┐     ┌─────────┐     ┌─────────────────────────┐
  │ blender python script  │ ──▶ │  .glb   │ ──▶ │ godot 4 import → scene  │
  │   bpy procedural mesh  │     │  binary │     │   custom gouraud shader │
  └────────────────────────┘     └─────────┘     └─────────────────────────┘
       (edit constants,             (vertex          (per-vertex Lambertian
        re-run, deterministic)       colors           term reads color attr;
                                     baked in)        cheap, mid-90s look)

Each script in godot/tools/blender/ is self-contained and deterministic. Run it; get the same mesh every time. Edit the constants at the top of a script, re-run, get the new mesh.

Output lands at godot/assets/3d/<category>/<asset>.glb. Godot's GLB import reads vertex colors as COLOR_0; the project's gouraud_lambert.gdshader uses that attribute as the lighting term.

REQUIREMENTS

  • Blender 3.6+ on PATH (or known absolute path).
  • No external Python dependencies beyond Blender's bundled bpy.
  • No DCC tool surface needed — Blender is a build target, not an authoring environment.

AESTHETIC RULES · LOCKED

POLYGON BUDGET

  • Cathedral interior: ~5-10k triangles total for the full walkable warehouse.
  • Individual props: 50-500 triangles each.
  • Agent figurines: 30-80 triangles each.
  • Regional dioramas (tabletop scale): 200-600 triangles each.
  • Full scene load target: 50ms on integrated graphics from a decade ago.

SHADING

  • Vertex colors carry the Gouraud Lambertian lighting term.
  • Bake-time light directions: cool-blue from (+X, 0, +Z) (the river-window side), warm bulb from (0, 0, +Z), ambient floor of (0.10, 0.08, 0.06).
  • Per-face vertex split for the slightly chunky mid-90s look — no smooth-shading across edges by default.
  • Materials are diffuse-only base + painted 2D textures applied via image-texture to specific faces.

LIGHTING (scene lights, in addition to vertex bake)

  • 1-3 lights per room.
  • Warm/cool contrast is the chief tool — never use white light.
  • Static interior: bake GI for the room, accept the cost.
  • Dynamic interior elements (BBS phosphor glow, workbench lamp): runtime point lights, low energy.

IMPORT SETTINGS (per GLB)

  • Generate Mipmaps: off (the painted textures don't need them; we want crisp pixel art at distance).
  • Compress: VRAM Compressed for surface textures; lossless for the BBS phosphor screen.
  • Use Named Skins: N/A (no skeletons).
  • Vertex Colors: enabled (this is how the Gouraud bake lands in the engine).

THE SHADER · gouraud_lambert.gdshader

shader_type spatial;
render_mode unshaded, cull_back;

void fragment() {
    ALBEDO = COLOR.rgb;       // the baked Gouraud lighting term
    // optional: layer a base diffuse map on top
    // ALBEDO *= texture(BASE_MAP, UV).rgb;
}

~10 lines of GDshader. Reads the per-vertex baked light term; ignores Godot's runtime lighting entirely for the "look" pass. Real-time lights (the BBS phosphor glow, the workbench lamp) are layered on top via additive secondary materials on specific faces.

SCENE ASSEMBLY

  • The Cathedral interior is one root scene · godot/scenes/cathedral.tscn.
  • Props instance individually for hot-reloadability.
  • The strategic-map dioramas are 3D children of the floor; the player walks up to inspect.
  • The BBS terminal triggers a fullscreen takeover scene (bbs_screen.tscn) when the player sits at it.

VISUAL REFERENCES

The aesthetic sits between mid-90s low-poly and modern restraint:

  • Quake (1996) — sector lighting, warm/cool contrast
  • Thief: The Dark Project (1998) — torch-lit interiors, deep blacks
  • LSD: Dream Emulator (1998) — surreal low-poly
  • Mouthwashing (2024) — modern PS1-aesthetic horror
  • Signalis (2022) — pixel-modern hybrid
  • Anatomy by Kitty Horrorshow — first-person low-poly minimalism
  • Cruelty Squad — aggressively striking with low fidelity
  • Paratopic, Iron Lung — vibe-led, low-spec, striking
